# Forkify QA Testing Project

## 📌 Project Overview
This repository contains manual QA testing results for the Forkify web application. The testing was performed to identify functional and UI issues through exploratory testing, along with basic verification using smoke tests.

## 🧪 Testing Type
- Manual Testing
- Exploratory Testing
- Functional Testing
- Smoke Testing

## 🐞 Bug Report Summary
- Total Bugs Reported: 18
- All bugs were logged and managed in Jira
- Bug reports are exported and included in CSV format in this repository

## 🧪 Smoke Testing
A smoke test was performed to verify the basic and critical functionalities of the Forkify application. The results are documented in a PDF file included in this repository.

## 🛠 Tools Used
- Jira (for bug tracking and reporting)
- Manual browser testing
- Smoke testing techniques

## 📁 Files Included
- Jira Bug Export (CSV)
  → Contains all bug reports logged in Jira with details like summary, description, severity, and steps to reproduce

- Smoke Test Report (PDF)
  → Contains smoke test cases and results verifying core functionality of the application

## 🎯 Objective
The objective of this project was to practice real-world QA testing by identifying and reporting defects in Jira, executing smoke tests, and documenting results in structured formats.

## 📌 Notes
This project is part of QA practice work to improve skills in manual testing, bug reporting, Jira usage, and test documentation.
